01 — Problem

The Problem to Solve

Lagos has millions of skilled informal workers — electricians, cleaners, movers, handymen — but no reliable digital infrastructure connecting them with people who need their services. The gap wasn't discovery, it was trust and workflow. People couldn't vet workers, and workers had no way to build reputation at scale.

The brief: build a platform that makes task-posting and worker-matching fast, trustworthy, and mobile-friendly for Lagos users.

02 — Stack & Tools

Tech Stack

Chosen for speed of deployment without sacrificing production quality. The stack needed to handle authentication, dynamic data, real-time job board updates, and a clean responsive UI.

ReactNode.jsREST APIAuthentication (JWT)CSS ModulesVercelClaude CodeCursor IDE
03 — Architecture

System Architecture

Architected as a client-server model with a React frontend consuming a Node.js REST API. Authentication flows use JWT tokens with refresh logic. The task board uses polling for near-real-time updates without WebSocket overhead at this scale.

Frontend

React SPA with component-based architecture. Task board, user profiles, and job posting flows built as isolated modules.

Backend

Node.js REST API handling task CRUD, user authentication, worker-task matching logic, and review aggregation.

Deployment

Frontend deployed to Vercel for edge delivery. API on Node environment. CI/CD via GitHub for zero-downtime deploys.

04 — AI-Assisted Workflow

AI-Accelerated Development

Claude Code was used to scaffold the initial API structure, generate boilerplate authentication middleware, and review complex matching logic for edge cases. Cursor handled in-context refactoring and component extraction in the React frontend.

The result: what would take a team 2–3 weeks to scaffold was compressed into a focused sprint. AI tools handled predictable code so engineering attention focused on the platform's core — the task-matching algorithm and trust/rating system.

05 — Challenges Solved

Key Engineering Challenges

Worker-task matching: Built a scoring system that weighs proximity, skills, rating, and availability. Simple enough to be fast, smart enough to surface the right workers.

Mobile-first for Lagos: Primary user base is on mobile, often on slower connections. Optimized for performance — lazy loading, minimal JS payloads, aggressive caching on the API.

Trust architecture: Designed a two-way review system with verified task completion before reviews unlock. Prevents gaming while encouraging honest feedback.
